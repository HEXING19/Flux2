# Flux XDR 智能安全运营平台 用户手册

## 一、 产品简介

### 1.1 产品定位
Flux XDR 是一款融合了大语言模型（LLM）与 XDR（Extended Detection and Response）能力的**企业级智能安全运营平台**。传统安全运营往往面临海量告警疲劳、系统割裂、多平台跳转切换成本高昂等问题，而 Flux 期望通过“自然语言交互”统一串联「事件研判、批量编排、处置审批、情报增强、核心资产视角」全流程，把告警处理从手工排查升级为**可复用、可审计的闭环运营**。

### 1.2 核心机制体系
- **自然语言执行框架 (AI Copilot)**：将意图精确转化为系统执行指令 (IR)，大幅降低运营人员的查询门槛和脚本编写负担。
- **高危行为熔断拦截 (Safety Gate)**：彻底屏蔽 AI 幻觉和误操作对于核心业务的影响，内置拦截白名单与人工二次审批。
- **场景剧本引擎 (Playbook Hub)**：沉淀专家经验，针对典型日常早报、深入研判、资产体检及溯源排查等场景提供编排好的一键自动化流。

## 二、 功能模块详解

### 2.1 工作台认证接入层 (Auth Gateway)
Flux 采用轻量化分布式控制端架构体系，用户需通过工作台认证才能建立通信。
- **控制中心地址配置**：对接企业既有的 XDR 端点 API（如 深信服 控制中心、本地平台后端接口等），支持 HTTPS 安全接入。
- **联动鉴权码**：利用 AK/SK 或专属授权码进行加密通讯绑定。提供 **探通测试** 以验证控制面通信正常。一旦通信建立，系统会快速加载之前的 LLM 对话上下文。

### 2.2 全局管控与设置 (Settings Management)
进入系统后，点击右上角的 **管理设置** 进入全局控制层面。主要包含四个子模块：
#### 2.2.1 LLM 认知引擎接入管理
- **功能描述**：配置 Flux 后台理解使用者意图的 AI 大脑。支持业内主流的大模型如 **OpenAI**、企业级 **智谱 AI**、国产性能前沿 **DeepSeek**，甚至支持通过 **第三方边界 (Custom)** 接入私有化部署的大模型。
- **配置项**：填写 API Key 密钥及模型标识符（如 `glm-4`、`gpt-4o` ），利用【路由探活】可确认当前所选模型网络及权鉴的有效性。

#### 2.2.2 高危指令硬拦截防线 (Safety Gate Rules)
- **功能描述**：用于设定 Flux Agent 在任何情况下都**绝对禁止封禁或破坏**的 IP/域名/网段。此模块直接解决自动化带来的业务中断顾虑。
- **配置项**：除系统内置的 `127.0.0.1` 环回口与局域网常规网关拦截外，管理员需添加如：数据中心核心汇聚、管理网络（如 `192.168.0.0/16`）的安全防线目标向量。只要匹配此清单，AI 所有的处置动作将被**硬性拒绝下发**。

#### 2.2.3 核心资产管理
- **功能描述**：维护企业最重要的保护对象台账（如 CRM主库、核心交易网关等）。
- **用途**：该台账不仅服务于人工备查，更能够被直接索引进 【核心资产防线透视 Playbook】中，实现资产级优先级的安全体检。

#### 2.2.4 威胁情报联动 (ThreatBook)
- **功能描述**：通过微步在线 (ThreatBook) API 拓展原生 XDR 研判能力不足的数据盲区。当平台对某个攻击源 IP 持疑时，系统将主动查询其云端情报置信度（恶意标签、团伙、C&C 控制等），显著强化封禁动作的判定依据。

---

### 2.3 AI 融合交互终端 (Intelligent Copilot Console)
您可以在主界面的聊天框内输入自然语言请求，这是平台最核心的操作入口。
- **自然语言查询与编排执行**
  支持极具口语化的指令，如：*“查询过去24小时内高位以上的安全告警事件”* 或 *“昨天有哪些来源于 1.2.3.4 的攻击？”* 系统会自动将您的描述转义并调用相应 API 获取结果卡片。
- **上下文序号追踪 (Context Mapping)**
  当您查询得到包含 5 条数据的告警列表时，无需复杂输入，继续对 AI 说：*“查看第一条告警的具体举证，并把第2个IP直接封掉”*。AI 会根据多轮缓存精确指向您正在操作的对象 UUID 及 IP。
- **防呆及缺参引导交互**
  如果您给出一条参数缺失的意图，例如：*“帮我把这个恶意IP封禁了”* 但却未表明封禁多久，AI 将**主动挂起任务**并追问用户：*“请问需要将该IP封禁多长时间（例如24小时、永远）？”*。这大大降低了操作失误可能性。

---

### 2.4 场景百宝箱 (Playbook Hub)
将高频且复杂的安全场景进行自动化封装组装，目前系统主要提供四大标准化模版剧本：
1. ☕ **今日安全早报 (Routine Check)**
   - **适用场景**：早安例会前、交接班时。
   - **执行过程**：系统自动提取过去 24 小时全部安全日志的总量起伏，抓取尚未被处置完成的 Top 高危事件数据，附上研判依据，为您生成排版精美的每日晨报总结。
2. 🔍 **单点告警深度研判 (Alert Triage)**
   - **适用场景**：面对个别极为模糊且高危的黑盒告警时。
   - **执行过程**：指定特定告警序号后，系统将围绕该目标做内网影响面追溯及外部情报穿透计算，判断是否为真实攻击还是误报跳版，并落出权威处置建议。
3. 🕵️ **攻击者活动轨迹生成 (Threat Hunting)**
   - **适用场景**：重保时期追溯特定 APT 黑客持续性行为。
   - **执行过程**：填入长期作恶者的源 IP，系统将回溯过去最长 90 天内（由外部传入内网）的所有触碰节点，串联形成从尝试试探到攻陷的横向时间线故事版。
4. 🏥 **核心资产一键体检 (Asset Guard)**
   - **适用场景**：关注重点业务系统是否存疑。
   - **执行过程**：关联核心资产台账 IP，做内部到外部、外部到内部的双向流量安全判定扫描，直观呈现该核心业务处于何种脆弱/遭受攻击频率。

---

### 2.5 特权操作人工复核与一键处置
平台严格区隔了**查询型能力**与**变更处置型能力**的边界。

- **人工安全红线审查 (Human-in-the-loop)**：当在交互终端请求下发封禁网侧防火墙等操作时，AI 将整理好其理解的“封禁对象、持续时间、操作理由”。接着必定弹出红色的 **特权操作二次熔断确认** 窗口，要求运营人员点检通过后才产生真实物理效应。
- **智能一键网侧联动封禁**：在“单兵告警戒点研判”或获取详情卡片中，您可以发现系统已提前通过微步测算出对应威胁 IP 的恶性权重，只要在“一键处置配置面板”中稍作滑动和校验，点击「确认」，即可瞬间调用深信服 AF 防火墙 API 将其阻断。

## 三、 最佳实战使用路径推荐

我们为企业运营团队提炼出了最优的标准日间安全流运转策略：

**Step 1: 平台启停与基建配置 (初始化阶段)**
首次部署完毕后，先进入【管理设置】，在 `大模型认知引擎` 里配置好稳定强大的底座（推荐配置有强大长序列推演能力的 DeepSeek 或者 OpenAI GPT-4o 系列），接着去验证 `威胁情报联动` 的 ThreatBook API 连通性，奠定高维研判基石。

**Step 2: 建立核心资产防线与白名单红线 (不可侵犯的准备)**
在做任何拦截操作前，去 `高危指令硬拦截防线` 页面加上各办公大楼 NAT 的出口节点 IP 及云端堡垒机 IP；并进入 `核心资产管理` 分门别类登记核心业务数据库。这样就完成了“AI 的护体金衣”。在此之后，放心下发操作，AI 不会因幻觉关停公司业务。

**Step 3: 日常运营首刀 —— 每日早报体检 (破晓期)**
每天上午刚坐下办公桌，直接打开 【Playbook : 今日安全早报】，系统运行半分钟后，你会对今天的整体外患内忧建立宏观视阈，直接得知当前迫切需要去处置那几个最高危卡点事件。

**Step 4: 告警溯源由面及点 —— 深度研判与轨迹重构 (研判期)**
在早报中看到的未解告警，比如编号为 2 的事件，直接在底部 AI 输入框输入：*“帮我详细看一下刚刚早报里第 2 个安全告警的具体证据”*。AI 给明细如果不足以判定定性，可以通过 【Playbook: 单点深度研判】 或进一步对黑客的 IP 运行 【Playbook: 攻击者活动轨迹生成】。一顿组合拳之后，您已掌握充足举证将其定性为“恶意攻击”或“误报”。

**Step 5: 安全闭环决断 —— AI 辅助审批降噪与拦截 (处置期)**
既然确认是必须拔除的毒刺，不再需要登录底层设备逐个去翻找访问控制（ACL）规则，直接回复：*“把刚刚查出来的这个黑客IP立刻阻断 24 小时，理由填入早报追溯恶意挂马”*。AI 弹窗出二次确认单进行人类审计，您只需用1秒钟审批点下“通过审计并下发”，业务就自动封堵并记录。整个繁杂的排查-取证-阻断过程至此完成降噪闭环。

## 四、 附录：核心交互例句参考

在主对话框您可以使用极度贴合人类习惯的话术：

- **场景挖掘类：**
  - *"看看昨天到现在有什么高危的外部攻击告警。"*
  - *"查一下 192.168.1.10 这台机器身上发生的相关事件有哪些？"*
- **追溯联动类（依赖上下文序号）：**
  - *"帮我看看刚才你发出来的表格里，第一条跟第三条到底都是怎么被攻击的？"*
  - *"把刚刚查询结果里面的第1个IP的外网状态及历史恶意记录拉取一下。"*
- **指令操作类：**
  - *"我确认刚才那个是挖矿软件产生的IP，帮我在内网防火墙上把它永远给封禁掉。"*
  - *"你帮我看看目前系统里面正在被封禁的IP清单有哪些？能不能帮我把最后一个误禁的解封。" (解封视底层 API 支持度)*
