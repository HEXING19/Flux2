### 阶段二：会话状态记忆与上下文映射机制 (M2)
**目标**：实现自然语义中指代词与真实底层 UUID 的桥接。

- [x] **2.1 ContextManager (状态管理器) 开发**
  - [x] 2.1.1 开发 `SkillContextManager` 类，实现按会话 Session ID 的状态隔离存储。
  - [x] 2.1.2 维护“参数与意图继承内存”：当当前对话依赖参数为空时，自动向上多轮历史追溯并回填。
- [x] **2.2 关联索引池 (Index Dictionary) 开发**
  - [x] 2.2.1 在 `ContextManager` 中实现列表数据的双向映射字典生成逻辑（`[序号前缀 / 指代符] <-> [业务UUID / 规则ID]`）。
  - [x] 2.2.2 完善意图解析逻辑：拦截带有“序号类”词汇的意图，自动转化为底层对应的真实业务变量后再下发被调用的 Skill。

### 阶段三：面向对象的 Skills 系统与安全 API 对接 (M3)
**目标**：基于架构规划落地各项原子能力，夯实“渐进式追问防线”。

- [x] **3.1 Skill 抽象积木 (Base Class System)**
  - [x] 3.1.1 定义 `BaseSkill` 抽象基类，规范包含 `__init_schema__` 与 `execute_api_requester` 等核心链路方法。
  - [x] 3.1.2 开发时间序列化中间件：拦截自然语言时间描述（如“最近三天”）并转化为系统标准 64位绝对时间戳（`startTimestamp`, `endTimestamp`）。
  - [x] 3.1.3 构建“渐进式追问防线”：定义 `MissingParameterException` 异常，当参数不足时主动停止执行流，指挥前端以自然语言向用户发起反问。
- [x] **3.2 安全事件管理 Skills 开发 (由 AI 直接编码实现)**
  - [x] 3.2.1 由 AI 助手编写 `EventQuerySkill`：根据文档实现条件查询，将提取的 UUID 写入上下文，组装前端所需的高亮表格结构。
  - [x] 3.2.2 由 AI 助手编写 `EventDetailSkill`：根据上下文 UUID 获取事件详情，拼装时间线、IP与举证实录。
  - [x] 3.2.3 由 AI 助手编写 `EventActionSkill`：接收批量 UUID 和状态字（0/10/40/60），完成安全操作代码逻辑。
- [x] **3.3 实体封禁管理 Skills 开发 (由 AI 直接编码实现)**
  - [x] 3.3.1 由 AI 助手编写 `BlockQuerySkill`：查询封禁状态并提取 `rule_id` 到上下文。
  - [x] 3.3.2 由 AI 助手编写 `BlockActionSkill (网侧执行)`：编写封禁核心防控逻辑，内置缺失参数时的反问防呆（防错）代码拦截。
  - [x] 3.3.3 由 AI 助手编写 `EntityQuerySkill`：编写获取项目 Config 并调用微步情报体系的接口逻辑代码。
- [x] **3.4 日志统计 Skill 开发 (由 AI 直接编码实现)**
  - [x] 3.4.1 由 AI 助手编写 `LogStatsSkill`：获取聚合数据并编写将其转化为标准 **Echarts Option JSON** 结构的代码，用于多模态图表渲染。

**目标**：串联独立原子化技能，打造高价值的 SecOps 闭环流水线。

- [x] **4.1 DAG 执行控制器开发**

  - [x] 4.1.2 添加能够触发上述 Pipeline 运行的 Cron 定时调度触发引擎。
- [x] **4.2 【每日严重高危闭环】主流程落地**
  - [x] 4.2.1 节点 1 开发：实现底层自动拉取“每日高危告警”的批量感知能力。
  - [x] 4.2.2 节点 2 开发：使用 DAG 并发串接 `EventDetailSkill` 和 `EntityQuerySkill`，实现异步深挖外部情报联动分析。
